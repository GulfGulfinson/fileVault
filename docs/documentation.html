Installation

Folgen Sie diesen Schritten, um FileVault auf Ihrem System zu installieren:

Voraussetzungen

  • Java 11 oder höher (empfohlen: Java 17 oder 21)
  • Maven für die Erstellung

Standardinstallation

git clone https://github.com/GulfGulfinson/fileVault.git
cd fileVault
mvn clean package
./start.sh   # Unter Linux/Mac
start.bat    # Unter Windows

Docker-Installation

docker pull ghcr.io/GulfGulfinson/fileVault:latest
docker run -v ~/.filevault:/root/.filevault ghcr.io/GulfGulfinson/fileVault:latest

Verwendung

Nach dem Start der Anwendung können Sie folgende Aktionen durchführen:

Erster Start

Bei der ersten Ausführung werden Sie aufgefordert, ein Passwort zu erstellen. Dieses Passwort wird zur Verschlüsselung Ihrer Dateien verwendet.

Ordner erstellen

Klicken Sie auf "Neuer Ordner" und geben Sie einen Namen ein, um einen virtuellen Ordner zu erstellen.

Dateien importieren

Wählen Sie einen Ordner aus und klicken Sie auf "Datei importieren", um eine Datei zu verschlüsseln und in den Tresor zu importieren.

Dateien exportieren

Wählen Sie eine Datei aus und klicken Sie auf "Exportieren", um die Datei zu entschlüsseln und an einem gewünschten Ort zu speichern.

Technische Details

Verschlüsselung

  • Algorithmus: AES-256-GCM (Galois/Counter Mode)
  • Schlüsselableitung: PBKDF2 mit HMAC-SHA256
  • Schlüssellänge: 256 Bit
  • Iterationen: 65.536
  • IV-Länge: 96 Bit
  • Authentifizierungs-Tag: 128 Bit

Datenspeicherung

Die verschlüsselten Dateien werden in ~/.filevault/data/ gespeichert, während die Metadaten in einer SQLite-Datenbank unter ~/.filevault/vault.db abgelegt werden.
